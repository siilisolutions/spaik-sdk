# 001: Add NoOpTraceSink and global tracing configuration

## Context

Currently, each agent creates its own trace sink via `get_trace_sink()`, which reads from an env var and defaults to writing to the filesystem. Library users cannot configure a custom trace sink globally - they must pass it to every agent constructor. This step introduces a module-level registry pattern (similar to Python's logging) that allows users to call `configure_tracing()` once at startup.

Additionally, the default behavior needs to change from writing to filesystem (which can be surprising) to doing nothing (no-op). The filesystem sink becomes opt-in via env var.

## Related Code

Existing files/patterns to reference or extend:
- `spaik_sdk/tracing/trace_sink.py` - The TraceSink abstract interface to implement
- `spaik_sdk/tracing/local_trace_sink.py` - Example TraceSink implementation to follow as pattern
- `spaik_sdk/tracing/get_trace_sink.py` - The factory function to modify with new resolution logic
- `spaik_sdk/tracing/__init__.py` - Module exports, needs to include new public function
- `spaik_sdk/tracing/trace_sink_mode.py` - Enum for sink modes, referenced in resolution

## Reuse Opportunities

- NoOpTraceSink follows the exact same interface as LocalTraceSink, just with empty implementation
- The module-level registry pattern is standard Python (like logging.setLevel) - no external dependencies needed
- get_trace_sink already handles the resolution logic, just needs additional checks in the chain

## Deliverables

- New NoOpTraceSink class implementing TraceSink interface with empty save_trace method
- Module-level variable to store the globally configured TraceSink
- New configure_tracing function that sets the global sink
- Updated get_trace_sink function with new resolution order: LOCAL env var → global sink → NoOpTraceSink
- Export configure_tracing from the tracing module's __init__.py

## Acceptance Criteria

- [ ] NoOpTraceSink exists and implements TraceSink interface
- [ ] Calling NoOpTraceSink.save_trace does nothing and does not throw
- [ ] configure_tracing function is exported from spaik_sdk.tracing
- [ ] After calling configure_tracing with a custom sink, get_trace_sink returns that sink
- [ ] When TRACE_SINK_MODE=local is set, get_trace_sink returns LocalTraceSink regardless of global config
- [ ] When TRACE_SINK_MODE=noop is set, get_trace_sink returns NoOpTraceSink regardless of global config
- [ ] When no config is set (no env var, no global sink), get_trace_sink returns NoOpTraceSink
- [ ] Calling configure_tracing multiple times replaces the previous sink (last one wins)
- [ ] Calling configure_tracing with None clears the global sink
- [ ] Unit tests verify all resolution scenarios
