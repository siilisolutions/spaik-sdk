# 005: Update env config defaults

## Context

Currently, `TRACE_SINK_MODE` defaults to "local" when the env var is not set, which means the SDK writes trace files to the filesystem by default. This is surprising behavior for library users who may not expect filesystem writes.

The new behavior should be: if `TRACE_SINK_MODE` is not set, return None/unset so that get_trace_sink can fall through to the global sink or no-op default. Only when explicitly set to "local" should LocalTraceSink be used.

This step completes the tracing configuration changes by ensuring the env var handling aligns with the new resolution logic from step 001.

## Related Code

Existing files/patterns to reference or extend:
- `spaik_sdk/config/env.py` - Contains get_trace_sink_mode method that needs to change its default behavior
- `spaik_sdk/tracing/trace_sink_mode.py` - Contains TraceSinkMode enum and from_name method that needs to handle empty/None values
- `spaik_sdk/tracing/get_trace_sink.py` - Already updated in step 001, but depends on env config returning None when unset

## Reuse Opportunities

- The pattern of returning None for optional/unset config already exists in other parts of env.py (e.g., required=False parameters)
- TraceSinkMode.from_name can follow similar pattern to other enum from_name methods that handle missing values

## Deliverables

- Update EnvConfig.get_trace_sink_mode to return None when TRACE_SINK_MODE env var is not set (instead of defaulting to "local")
- Update TraceSinkMode.from_name to handle empty string and None inputs gracefully by returning None
- Ensure get_trace_sink (from step 001) properly handles None return from get_trace_sink_mode

## Acceptance Criteria

- [ ] When TRACE_SINK_MODE env var is not set, get_trace_sink_mode returns None
- [ ] When TRACE_SINK_MODE env var is empty string, get_trace_sink_mode returns None
- [ ] When TRACE_SINK_MODE env var is "local", get_trace_sink_mode returns TraceSinkMode.LOCAL
- [ ] When TRACE_SINK_MODE env var is "noop", get_trace_sink_mode returns TraceSinkMode.NOOP
- [ ] TraceSinkMode.from_name returns None for empty string input
- [ ] TraceSinkMode.from_name returns None for None input
- [ ] TraceSinkMode.from_name returns LOCAL for "local" input (existing behavior)
- [ ] TraceSinkMode.from_name returns NOOP for "noop" input
- [ ] End-to-end: fresh install with no env vars results in no-op tracing (no filesystem writes)
- [ ] End-to-end: setting TRACE_SINK_MODE=local results in filesystem trace writes
- [ ] Unit tests verify all input combinations for get_trace_sink_mode and TraceSinkMode.from_name
